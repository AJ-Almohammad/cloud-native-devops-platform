1. Prerequisites

Ensure you have installed:

  • ‚úÖ Docker Desktop with Kubernetes enabled
  • ‚úÖ kubectl command line tool
  • ‚úÖ Project cloned locally

2. Build Microservice Images

Run these commands in your terminal:

# Build all microservices docker build -t content-api:local services/content-api/ docker build -t user-service:local services/user-service/ docker build -t analytics-service:local services/analytics-service/ docker build -t notification-service:local services/notification-service/ docker build -t cron-scheduler:local services/cron-scheduler/

3. Deploy to Kubernetes

# Create namespace and deploy kubectl create namespace multi-everything kubectl apply -f kubernetes/secrets.yaml -n multi-everything kubectl apply -f kubernetes/applications/ -n multi-everything # Verify deployment kubectl get pods -n multi-everything

4. Set Up Port Forwarding (CRITICAL)

üö® Important: You need to run these commands in 5 separate terminal windows to make all services accessible.

Terminal 1 - Content API
kubectl port-forward -n microservices service/content-api 8082:80
Terminal 2 - User Service
kubectl port-forward -n multi-everything service/user-service 8002:8000
Terminal 3 - Analytics Service
kubectl port-forward -n multi-everything service/analytics-service 3000:3000
Terminal 4 - Notification Service
kubectl port-forward -n multi-everything service/notification-service 8081:8080
Terminal 5 - Cron Scheduler
kubectl port-forward -n multi-everything service/cron-scheduler 8003:8000

5. Test Your Setup

Once all port-forwarding is running, switch back to the Overview tab. All 5 services should show as ‚úÖ HEALTHY.
